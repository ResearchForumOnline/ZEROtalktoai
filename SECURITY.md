# Security Policy

ZEROtalktoai is a public project hub for the TalkToAI and ResearchForumOnline ecosystem. It is not a production secrets store, deployment notebook, customer data repository, or private operations log.

## Supported Scope

Security reports are in scope when they relate to public project documentation, public links, public web pages, or public repository content connected to this hub.

For server hardening and update automation, use the ZSEC project:

- https://github.com/ResearchForumOnline/ZSEC
- https://talktoai.org/zsec/

## Do Not Publish Sensitive Material

Please do not open public issues or pull requests containing:

- SSH private keys, API keys, tokens, passwords, cookies, or session data.
- Private server login details, root access notes, DNS provider credentials, or panel credentials.
- Customer data, email exports, internal migration notes, or unpublished production code.
- Working exploit chains against live infrastructure.

If sensitive information is discovered, report it through the public contact route at https://talktoai.org/ and keep the details private until they can be handled safely.

## Responsible Disclosure

A useful report includes:

- The affected public URL, repository, or file path.
- A short description of the risk.
- Minimal reproduction steps that do not damage systems or expose private data.
- Suggested mitigation when known.

Reports that help improve documentation boundaries, broken links, public security guidance, and ZSEC hardening coverage are welcome.

## Project Boundary

This repository is intended to make the public project ecosystem easier to understand. Operational secrets and private infrastructure details belong outside GitHub.
